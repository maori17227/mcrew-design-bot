# Инструкция по ручному деплою на Cloudflare Pages

## Проблема
Твой сайт `mcrew-website.pages.dev` показывает старую версию (коммит `38f3107a`) вместо последней версии с карточкой MC Bank (коммит `d5bc8fa`).

## Почему это происходит
Автоматический деплой с GitHub на Cloudflare Pages не работает. Нужно вручную запустить деплой.

## Решение: Ручной деплой

### Шаг 1: Открой панель Cloudflare
1. Зайди на https://dash.cloudflare.com/
2. Войди в свой аккаунт
3. Нажми на **Workers & Pages** в левом меню
4. Найди и нажми на **mcrew-website**

### Шаг 2: Создай новый деплой
1. Нажми кнопку **"Create deployment"** (или **"Deploy"**)
2. В настройках деплоя:
   - **Branch** (Ветка): Выбери `main`
   - **Build command**: Оставь пустым или как есть
   - **Build output directory**: `website` или `/website`
3. Нажми **"Save and Deploy"**

### Шаг 3: Дождись завершения
- Деплой займет 1-2 минуты
- Увидишь прогресс-бар
- Когда закончится, появится "Deployment successful"

### Шаг 4: Проверь изменения
1. Открой свой сайт: https://mcrew-website.pages.dev
2. Жесткое обновление: Нажми `Ctrl + Shift + R` (Windows) или `Cmd + Shift + R` (Mac)
3. Теперь должно быть видно:
   - ✅ Карточка MC Bank с балансом
   - ✅ Формат баланса: "0.0" вместо "0"
   - ✅ Курс TON обновляется автоматически
   - ✅ Все последние исправления

## Альтернатива: Включить автоматический деплой

Чтобы в будущем деплой работал автоматически:

1. В панели Cloudflare → **mcrew-website** → **Settings**
2. Перейди в **Builds & deployments**
3. Проверь что **"Production branch"** установлена на `main`
4. Включи **"Automatic deployments"**
5. Убедись что интеграция с GitHub подключена

Если ручной деплой не работает:
1. Проверь что у тебя есть права в Cloudflare
2. Убедись что GitHub репозиторий подключен
3. Попробуй отключить и заново подключить интеграцию с GitHub
